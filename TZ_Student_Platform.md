# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Платформа поиска студентов-помощников "StudentHelper"

---

## 1. ОПИСАНИЕ ПРОЕКТА

**Наименование:** Платформа для поиска студентов-помощников по предметам

**Цель:** Создание веб-приложения, которое позволяет студентам младших курсов найти помощь у старшекурсников по учебным дисциплинам. Платформа работает как маркетплейс услуг между студентами.

**Целевая аудитория:** Студенты ВУЗа (все курсы)

**Срок реализации:** Демонстрационная версия

---

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1 Авторизация и профиль пользователя

- **Вход в систему:**
  - Номер телефона
  - Пароль
  - Выбор группы (факультета)
  - Выбор курса обучения
  - Кнопка "Войти"

- **Данные не требуют сохранения** (демо-версия)
- После успешной авторизации пользователь направляется на главную страницу

### 2.2 Каталог студентов

- **Отображение карточек студентов-помощников:**
  - Фотография (аватар)
  - Фамилия и имя
  - Год обучения (курс)
  - Направление/группа
  - Список предметов, по которым может помочь

- **Структура каталога как маркетплейс:**
  - Горизонтальное меню-фильтр вверху страницы
  - Кнопки фильтров: "1 курс", "2 курс", "3 курс", "4 курс", "5 курс", "Выпускники"
  - По умолчанию отображаются все студенты

### 2.3 Фильтрация и поиск

- **Фильтр по курсам:**
  - Активная кнопка выбранного курса (подсвечена)
  - При нажатии на кнопку курса - список обновляется
  - Можно выбрать несколько курсов одновременно (опционально)

- **Фильтр по предметам:**
  - Доступен на странице каталога
  - Возможность поиска/выбора по дисциплинам

### 2.4 Профиль студента

- **Детальная страница студента (при клике на карточку):**
  - Увеличенная фотография
  - Полная информация: ФИ, курс, группа
  - Полный список предметов, по которым оказывается помощь
  - **Контактная информация:**
    - Номер телефона
    - Ссылка на ВКонтакте (или другой мессенджер)
    - Email
  - Кнопка "Назад" для возврата в каталог

### 2.5 Генерация данных

- **Для демо-версии:**
  - Использовать нейросеть (AI) для генерации данных
  - Все данные выдуманные, но логически связанные:
    - Фамилии и имена студентов
    - Фото (сгенерированные изображения или плейсхолдеры)
    - Предметы в соответствии с курсом обучения
    - Вымышленные контакты

## 5. ПОЛЬЗОВАТЕЛЬСКИЕ СЦЕНАРИИ

### Сценарий 1: Студент ищет помощь по математике
1. Заходит на платформу → вводит данные авторизации
2. Видит каталог студентов
3. Выбирает фильтр "5 курс" (нужен опытный)
4. Видит карточки пятикурсников
5. Нажимает на карточку студента, который преподает математику
6. Открывается профиль с контактами
7. Звонит или пишет в ВК

### Сценарий 2: Четверокурсник фильтрует по курсам
1. На главной странице видит кнопки фильтра
2. Нажимает "3 курс" → видит только третьекурсников
3. Нажимает "5 курс" → видит только пятикурсников
4. Кнопка остаётся активной (подсвечена)

### Сценарий 3: Демонстрация преподавателю
1. Препод заходит на сайт (вводит любые данные)
2. Видит полностью заполненный каталог со студентами
3. Может кликать на студентов и смотреть профили
4. Может менять фильтры

## 6. ТРЕБОВАНИЯ К ДИЗАЙНУ И UX

### 6.1 Визуальное оформление
- **Цветовая схема:** Синий (основной) + белый + серый
- **Шрифт:** Arial, Open Sans или аналогичный sans-serif
- **Отступы:** Минимум 16px между элементами
- **Скругления:** Углы карточек скруглены (4-8px)
- **Тени:** Мягкие тени под карточками для глубины

### 6.2 Адаптивность
- **Desktop:** 3 колонки карточек
- **Tablet:** 2 колонки
- **Mobile:** 1 колонка

### 6.3 Интерактивность
- **Hover эффекты:**
  - На карточках: легкое увеличение, тень усиливается
  - На кнопках: изменение цвета, курсор pointer
- **Активные фильтры:** Подсвечены другим цветом (например, более темная синь)
- **Переходы:** Плавные (200-300ms)

## 7. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 7.1 Стек технологий

**Frontend:**
- HTML5
- CSS3 (или Tailwind/Bootstrap)
- JavaScript (vanilla или React/Vue)
- Возможно: React для интерактивности

**Backend (опционально для демо):**
- Node.js + Express (если нужна динамическая генерация)
- Python Flask/Django (если используется AI для генерации)

**База данных (опционально):**
- Для демо: статические данные в JSON
- Для production: PostgreSQL или MongoDB

**AI (для генерации данных):**
- OpenAI API (ChatGPT для текста)
- Generative AI для фото (можно использовать Lorem Picsum, Unsplash API)

### 7.2 Хостинг
- **Для демо:** GitHub Pages, Vercel, Netlify, Tilda
- **Для боевого:** VPS (DigitalOcean, AWS, Heroku)

### 7.3 Требования к производительности
- Загрузка страницы: < 3 секунд
- Переключение между курсами: < 500ms
- Открытие профиля: < 1 сек
